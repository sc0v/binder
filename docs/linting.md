# Linting

Binder uses four linters, all orchestrated through Rake:

| Linter | Covers | Config |
|--------|--------|--------|
| [Prettier](https://prettier.io) | JS, CSS, YAML, Ruby, ERB | `.prettierrc`, `.prettierignore` |
| [RuboCop](https://rubocop.org) | Ruby | `.rubocop.yml` |
| [ERBLint](https://github.com/Shopify/erb-lint) | ERB templates | `.erb-lint.yml` |
| [Brakeman](https://brakemanscanner.org) | Security (Ruby/Rails) | none |

## Running Linters

Run all linters with autocorrection:

```
bin/rails lint
```

Run all linters in check-only mode (no file modifications, what CI does):

```
bin/rails lint:dryrun
```

Run a single linter:

```
bin/rails lint:rubocop
bin/rails lint:prettier
bin/rails lint:erblint
bin/rails lint:brakeman
```

Run a single linter in check-only mode (via `rake`, not `bin/rails`):

```
rake lint:rubocop:dryrun
rake lint:prettier:dryrun
```

Lint a single file:

```
bin/rails lint[path/to/file.rb]
```

## Prettier

Prettier formats JS, CSS, YAML, Ruby (via `@prettier/plugin-ruby`), and ERB
(via `@4az/prettier-plugin-html-erb`).

### htmlWhitespaceSensitivity

`.prettierrc` sets `htmlWhitespaceSensitivity: "strict"`. This is required
because the CMU sidebar element is whitespace-sensitive. Prettier's default
behavior inserts a newline before block-level opening tags, which breaks the
sidebar layout.

The sidebar div must be written directly adjacent to the closing div before it,
with no whitespace between:

```erb
  </div><div class="sidebar">
```

If Prettier reformats this onto a new line the sidebar will not render
correctly.

### .prettierignore

Generated and upstream files are excluded from formatting:

- `db/schema.rb`: generated by Rails
- `Gemfile.lock`, `package-lock.json`: generated lockfiles
- `app/assets/stylesheets/cmu.css`: upstream minified file, not authored here
- `public/assets/`, `public/packs/`: compiled assets

## RuboCop

RuboCop runs with three extensions:

- `rubocop-rails`: Rails-specific cops
- `rubocop-minitest`: Minitest-specific cops
- `rubocop-capybara`: Capybara-specific cops

Notable configuration in `.rubocop.yml`:

- `Layout/LineLength: Max: 120`: Rails default of 80 is too narrow for this codebase
- `Style/Documentation: Enabled: false`: class-level doc comments are not required
- `Style/ClassAndModuleChildren: Enabled: false`: nested module style is not enforced

## ERBLint

ERBLint runs after Prettier as a prerequisite. It enforces:

- Final newline at end of file
- No trailing whitespace
- RuboCop rules applied to Ruby inside ERB tags

## Brakeman

Brakeman scans for security vulnerabilities in Ruby and Rails code. It does not
autocorrect. All warnings require manual review.

## CI

The CI workflow (`.github/workflows/ci.yml`) runs `bin/rails lint:dryrun` on
every push and pull request. PRs cannot merge with lint failures.
